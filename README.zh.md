# generate_assets

[English](README.md) | [中文](README.zh.md)

一个 Dart CLI 工具，从 Flutter 项目的 `pubspec.yaml` 生成资源常量文件，支持未使用资源检测与批量删除。

## 安装

在 Flutter 项目的 `pubspec.yaml` 中添加 `dev_dependencies`：

```yaml
dev_dependencies:
  generate_assets: ^0.1.0
```

然后执行：

```bash
dart pub get
```

## 命令

```bash
# 生成 lib/generated/assets.dart（或配置的输出路径）
dart run generate_assets

# 扫描 lib/*.dart，列出未使用的资源
dart run generate_assets find-unused

# 交互式删除未使用的资源（含 2x/3x 分辨率变体）
dart run generate_assets delete-unused
```

## 配置

在 `pubspec.yaml` **根节点**添加 `flutter_generate_assets` 配置：

```yaml
flutter_generate_assets:
  output: lib/common/assets.dart   # 默认：lib/generated/assets.dart
  class_name: Assets               # 默认：Assets

flutter:
  assets:
    - assets/images/
    - assets/icons/
```

## 生成结果示例

```dart
// GENERATED CODE - DO NOT MODIFY BY HAND
// Generated by generate_assets
// https://github.com/ceeyang/generate_assets
// ignore_for_file: lines_longer_than_80_chars, constant_identifier_names
// dart format off
class Assets {
  static const String imagesLogo = 'assets/images/logo.png';
  static const String imagesBgHome = 'assets/images/bg_home.jpg';
  static const String iconsArrowLeft = 'assets/icons/arrow-left.svg';
}
```

同名不同扩展名的文件，自动追加扩展名后缀以区分：

```dart
static const String copyToClipboardPng = 'assets/copy_to_clipboard.png';
static const String copyToClipboardSvg = 'assets/copy_to_clipboard.svg';
```

## 命名规则

- 路径分隔符、`_`、`-`、`.` 及空格均作为分词符
- 变量名为 **小驼峰（lowerCamelCase）**，包含目录层级（`assets/icons/arrow_left.svg` → `iconsArrowLeft`）
- 自动忽略 `2x`、`3x`、`1.5x` 等分辨率变体目录
- 自动忽略 `.DS_Store` 及以 `.` 开头的隐藏文件
- 重复路径自动去重

## 未使用资源检测

`find-unused` 扫描 `lib/` 下所有 `.dart` 文件（排除生成文件本身），检测两种引用方式：

1. 字符串字面量 — `'assets/icons/logo.svg'`
2. 常量引用 — `Assets.iconsLogo`

> **注意：** 字符串插值（如 `'assets/$name.png'`）无法静态检测，可能被误报为未使用。

`delete-unused` 在删除前会显示确认提示，列出所有将被删除的文件（含 `2x/3x` 变体）。

## 环境要求

- Dart SDK >= 3.0.0
- Flutter 项目，且 `pubspec.yaml` 位于当前工作目录根节点
